## 二、Sprint 1 — 补齐核心闭环（第 1 周）

目标：打通"面试 → 错题 → 学习计划"断裂链路，让用户可以走完完整的业务闭环。

### Day 1-2：Interview 模块后端真实实现 ✅

**任务清单：**

- [x] 新建 `InterviewService` 接口和 `InterviewServiceImpl` 实现类
- [x] 实现 `start()`：根据方向从题库抽取 N 道题（默认 3-5 题），创建 `interview_session` 记录，返回第一道题
- [x] 实现 `current()`：根据 sessionId 查询当前题目（通过 `current_index` 定位）
- [x] 实现 `answer()`：
  - 调用 `AiOrchestratorService.scoreInterviewAnswer()` 获取评分
  - 解析 LLM 返回的结构化 JSON（分数、点评、标准答案、追问）
  - 写入 `interview_record` 表
  - 如果分数低于阈值（如 60 分），自动写入 `wrong_question` 表
  - 更新 `interview_session.current_index`
  - 如果已是最后一题，更新 session 状态为 `finished`，计算 `total_score`
- [x] 实现 `detail()`：查询 session 关联的所有 record，组装为 `InterviewDetailVO`
- [x] 修改 `InterviewController`，注入 `InterviewService`，替换硬编码实现
- [x] 优化 `DefaultAiOrchestratorService.scoreInterviewAnswer()`：
  - 构建结构化 Prompt，要求 LLM 返回 JSON 格式：`{"score": 85, "comment": "...", "standardAnswer": "...", "followUp": "..."}`
  - 使用 Jackson 解析 LLM 返回的 JSON，填充 `InterviewAnswerVO`
  - 移除固定分数 `78` 和硬编码占位文本

**关键接口变更：**

```
POST /api/interview/start    → 真实抽题，创建 session
GET  /api/interview/current/{sessionId} → 从 DB 查询当前题目
POST /api/interview/answer   → AI 评分 + 持久化 + 自动入错题
GET  /api/interview/detail/{sessionId} → 查询面试记录详情
```

## 六、风险与应对

| 风险 | 概率 | 影响 | 应对方案 |
|------|------|------|---------|
| LLM 结构化输出解析失败 | 中 | 面试评分不可用 | 增加重试 + 兜底解析（正则提取分数）+ 默认评分 |
| Interview 抽题逻辑与题库数据不匹配 | 中 | 面试无法开始 | 先用简单随机抽题，确保题库有足够种子数据 |
| 前端改造工作量超预期 | 中 | Sprint 1 延期 | Interview/Wrong/Plan 优先保证功能可用，UI 细节放到 Sprint 3 |
| Redis 不可用导致 JWT 黑名单失败 | 低 | 登出功能降级 | 黑名单检查加 try-catch，Redis 不可用时跳过检查（降级为当前行为） |
| 全文索引对中文支持不佳 | 中 | 知识检索质量下降 | MySQL 8.x 支持 ngram 解析器，配置 `ngram_token_size=2` |
