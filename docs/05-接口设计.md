# ByteCoach 接口设计

## 1. 设计原则

- 所有接口统一前缀：`/api`
- 所有接口统一返回：`Result<T>`
- 认证接口与业务接口分离
- 读接口尽量 REST 化
- AI 相关接口保留明确的业务语义，不直接暴露底层模型调用

## 2. 统一返回结构

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

## 3. 鉴权与用户接口

### 3.1 注册

- `POST /api/auth/register`

请求体：

```json
{
  "username": "demo",
  "password": "123456",
  "nickname": "ByteCoach"
}
```

### 3.2 登录

- `POST /api/auth/login`

响应示例：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "token": "jwt-token",
    "userInfo": {
      "id": 1,
      "username": "demo",
      "nickname": "ByteCoach"
    }
  }
}
```

### 3.3 当前用户

- `GET /api/user/me`

### 3.4 退出登录

- `POST /api/auth/logout`

## 4. Dashboard 接口

### 4.1 首页概览

- `GET /api/dashboard/overview`

返回建议包含：

- 学习总次数
- 平均得分
- 错题数量
- 最近面试结果
- 薄弱分类 Top N
- 当前计划完成率

## 5. Chat 接口

### 5.1 发送消息

- `POST /api/chat/send`

请求体：

```json
{
  "sessionId": 1,
  "mode": "rag",
  "message": "Spring AOP 的底层实现原理是什么？"
}
```

响应体：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "sessionId": 1,
    "answer": "回答内容",
    "references": [
      {
        "docId": 1,
        "docTitle": "Spring 核心笔记",
        "chunkId": 101,
        "snippet": "引用片段"
      }
    ]
  }
}
```

### 5.2 会话列表

- `GET /api/chat/sessions`

### 5.3 会话消息

- `GET /api/chat/messages/{sessionId}`

### 5.4 删除会话

- `DELETE /api/chat/session/{sessionId}`

### 6.1 文档列表

- `GET /api/knowledge/list`

- `POST /api/knowledge/search`

请求体：

```json

  "query": "JVM 垃圾回收器分类"

```

### 6.3 后台导入文档

### 6.4 重新切分文档

- `POST /api/admin/knowledge/rechunk/{docId}`

### 6.5 重建向量索引

- `POST /api/admin/knowledge/reindex/{docId}`

## 7. 面试接口

### 7.1 开始面试

- `POST /api/interview/start`

请求体：

```json
{
  "direction": "Spring",
  "questionCount": 3
}
```

### 7.2 获取当前题目

- `GET /api/interview/current/{sessionId}`

### 7.3 提交答案

- `POST /api/interview/answer`

请求体：

```json
{
  "sessionId": 1,
  "questionId": 1001,
  "answer": "我的回答内容"
}
```

响应建议包含：

- 当前题得分
- AI 点评
- 标准答案
- 单次追问
- 是否加入错题本
- 是否还有下一题

### 7.4 面试记录详情

- `GET /api/interview/detail/{sessionId}`

## 12. 接口设计结论

ByteCoach 一期接口应突出“业务语义明确、闭环完整、AI 能力可扩展”三个特点，而不是直接暴露底层模型和检索细节。只要先把用户、首页、问答、面试、错题、计划这几组接口定义清楚，后续研发就能稳步展开。
