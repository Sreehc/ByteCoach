# OfferPilot PRD

## 文档说明

本文档描述 OfferPilot 当前正式产品能力，作为产品、研发、测试和运营协作的统一口径。

- 当前正式能力以本文件为准
- 前端页面体验审查任务统一维护在 [docs/future/README.md](../future/README.md)
- 本文档只描述当前已经对外成立的能力，不记录占位接口或规划态方案

## 产品定位

OfferPilot 是一个面向 Java 求职者的 AI 面试训练与求职管理平台。产品主线不是单一刷题，也不是单一 RAG 问答，而是把题库训练、知识问答、模拟面试、学习计划、简历打磨和投递管理组织成一条连续工作流。

## 目标用户

- 准备 Java 实习、校招、初级社招岗位的学习者
- 已有少量项目经验，但缺少系统化面试训练路径的开发者
- 需要同时管理知识资料、训练记录、简历版本和岗位投递进展的求职者

## 核心价值

- 主线统一：首页直接进入求职训练工作台，而不是在孤立工具页之间切换
- 训练结构化：题库、问答、面试、计划和简历围绕岗位准备形成连续闭环
- 资料可消费：知识文档、简历文件和训练记录能沉淀为后续问答、复盘和追踪素材
- 治理可见：后台可查看 AI 调用日志、系统配置、面试治理和内容状态

## 主闭环

```text
进入工作台
  -> 题库训练定位知识盲点
  -> 知识库上传/检索与 RAG 问答补齐理解
  -> 模拟面试验证表达、追问和答题质量
  -> 学习计划安排 7/14/30 天任务
  -> 简历助手整理项目经历、自我介绍和面试简历提纲
  -> 投递管理跟踪 JD、状态流转与面试事件
  -> 数据分析与后台治理辅助复盘
```

## 产品表达原则

- 页面首屏必须让用户快速判断“现在做什么”
- 页面标题、CTA、空状态和提示文案必须面向用户任务
- 主文案不使用系统自述、研发占位或阶段说明语气
- 同一页面首屏只保留一套主标题语义和一个主动作
- 功能再多，也要通过主次层级表达，而不是并列堆叠

## 功能结构

### 主线模块

| 模块 | 角色 | 当前能力 |
|------|------|----------|
| 首页工作台 | 训练入口 | 展示训练概览、最近面试、计划/简历/投递摘要与快捷动作 |
| 题库训练 | 结构化训练底座 | 支持分类、题型、难度、岗位方向与标签筛选，查看题目详情 |
| 知识库 | 资料中心 | 支持平台资料浏览、个人文档上传、解析、检索和删除 |
| RAG 问答 | 即时补疑 | 支持普通问答、资料问答、流式回答、引用来源与上下文问答 |
| 模拟面试 | 能力验证 | 支持文字/语音模式、历史记录、详情复盘、低分题卡片化 |
| 学习计划 | 节奏安排 | 支持生成 7/14/30 天计划、每日任务、任务状态更新、计划刷新 |
| 简历助手 | 求职材料整理 | 支持简历上传解析、项目拆分、项目追问、自我介绍、面试简历提纲 |
| 投递管理 | 求职流程跟踪 | 支持岗位录入、JD 分析、状态流转、事件记录、详情查看 |
| 数据分析 | 训练洞察 | 展示训练趋势、求职准备趋势与学习洞察 |
| 管理后台 | 内容与系统治理 | 提供用户、分类、题库、文档、AI 日志、系统配置、面试治理等后台能力 |

### 辅助模块

| 模块 | 角色 | 当前能力 |
|------|------|----------|
| 卡片强化 | 辅助巩固 | 支持 deck、今日任务、评分与激活 |
| 复习中心 | 间隔复习 | 基于 SM-2 的复习队列、评分和统计 |
| 错题本 | 面试/训练沉淀 | 管理错题、掌握状态和 Markdown 导出 |
| 社区 | 外部交流 | 支持提问、回答、投票和排行榜 |
| 设置与安全 | 账户管理 | 支持头像、设备管理、登录历史、邮箱验证、找回密码、2FA、数据导出 |

## 当前正式能力清单

### 账号与安全

- 用户注册、登录、登出
- 图形验证码与登录日志
- 邮箱验证码验证与找回密码
- 设备管理与撤销
- 两步验证和恢复码
- 头像上传与个人信息查询

### 训练与学习

- 题库分页查询、标签筛选与题目详情
- 知识文档上传、平台资料浏览和向量检索
- 支持带引用的知识问答与流式响应
- 支持简历或项目上下文的问答链路
- 文字/语音模拟面试、历史、详情与趋势
- 7/14/30 天学习计划生成、刷新和任务打卡
- 错题本与 SM-2 复习队列

### 求职材料与投递

- 简历上传解析、项目拆分、项目追问、自我介绍、面试简历提纲
- 岗位投递看板、详情页、JD 分析、状态流转和事件流

### 治理与运营

- 管理后台系统概览、用户管理、内容审核
- 分类管理、题库管理、文档管理
- AI 调用日志与系统配置覆盖
- 面试治理与全局登录日志
- 题库/用户导出与题目批量导入

## 成功标准

- 新用户进入首页后，能在短时间内理解“从训练到简历到投递”的主线
- 首页首屏能让用户在几秒内判断当前主任务和下一步动作
- 题库、问答、面试、计划、简历、投递六个核心模块都能独立进入并形成可用链路
- 页面标题、按钮、空状态和提示文案都以用户动作和结果为中心
- 用户可以从知识资料、训练记录和投递事件中持续积累求职准备材料
- 管理员可以在后台查看内容状态、AI 调用状态和系统配置，而不是依赖数据库直查

## 当前边界

- 正式产品说明不包含第三方 OAuth 完整登录链路
- 存储当前以统一抽象 + 本地文件存储实现为准
- 正式 PRD 不展开前端页面审查过程与问题清单，统一见 [docs/future/README.md](../future/README.md)
