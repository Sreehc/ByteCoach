# ByteCoach 产品总览

## 项目定位

ByteCoach 是一个面向 Java 后端学习者的记忆驱动学习平台。核心不是把用户分流到问答、面试、分析等独立模块，而是先把资料转成每天可完成的卡片任务，再通过复习、问答和面试诊断形成持续强化。

## 核心价值

- **主线明确**：用户进入产品后先完成今日卡片，不需要在多个学习模块之间做选择
- **间隔记忆**：系统基于记忆曲线持续安排待学、待复习、完成率和连续学习节奏
- **资料可转化**：知识库中的文档会被自动加工成可执行的记忆任务，而不是停留在被动阅读
- **辅助闭环**：问答用于澄清，面试诊断用于验证，分析用于复盘，三者都服务于卡片主线

## 主闭环

知识资料 -> 生成卡片 -> 完成今日卡片 -> 进入复习中心 -> 用问答补疑问 -> 用面试诊断做进阶验证 -> 数据分析复盘 -> 回到下一轮卡片

## 功能模块优先级

### 一期核心：卡片记忆系统

| 模块 | 功能 | 说明 |
|------|------|------|
| 今日卡片 | 资料转卡片 | 从知识库文档中自动抽取问答卡片 |
| 今日卡片 | 按天推进 | 任务按天分配待学卡片，形成固定日任务 |
| 今日卡片 | 卡片评分 | 通过重来 / 困难 / 良好 / 轻松推进状态 |
| 今日卡片 | 掌握跟踪 | 统计待学、待复习、完成率、已掌握卡片 |
| 复习中心 | 到期复习 | 统一处理到期复习和复盘积压 |
| 复习中心 | 间隔调度 | 基于 SM-2 计算下次复习日期与掌握状态 |
| 首页 | 任务看板 | 首屏只显示今天该完成的记忆任务与进度 |

### 二期核心：资料与辅助理解

| 模块 | 功能 | 说明 |
|------|------|------|
| 知识库 | 文档管理 | 上传/导入学习资料（PDF/TXT/MD） |
| 知识库 | 自动切分 | 文档按 token 数自动分片 |
| 知识库 | 向量索引 | Embedding 向量化 + Redis 向量搜索 |
| 智能问答 | 普通问答 | 基于 LLM 的 Java 知识问答 |
| 智能问答 | 资料问答 | 基于知识库上下文回答，支持引用溯源 |

### 三期核心：能力验证与分析

| 模块 | 功能 | 说明 |
|------|------|------|
| 面试诊断 | 文字诊断 | 3-5 题/场，AI 实时评分、点评、追问 |
| 面试诊断 | 语音诊断 | Whisper STT 转写 + TTS 播放，支持语音作答 |
| 面试诊断 | 诊断历史 | 查看所有已完成诊断的详情和复盘 |
| 数据分析 | 趋势分析 | 按周统计学习趋势、掌握分布和效率变化 |
| 数据分析 | 记忆分析 | 查看完成率、记忆强度、遗忘率与积压情况 |

### 支撑模块

| 模块 | 功能 | 说明 |
|------|------|------|
| 社区 | 提问 / 回答 / 排行榜 | 作为外部交流与补充，不承担学习主线 |
| 管理后台 | 用户 / 内容 / 文档管理 | 提供内容运维与系统配置能力 |
| 账户与安全 | 登录 / 设备 / 2FA / 导出 | 提供可生产使用的账户安全基础设施 |

## 目标用户

- 准备 Java 后端面试、但不希望只靠刷题突击的学习者
- 有资料积累、希望把资料转成稳定记忆任务的工程师
- 需要长期复习节奏，而不是靠临时冲刺的学习者

## 一期成功标准

- 用户进入首页后，能在 3 秒内理解今天先做什么
- 首页第一按钮始终指向今日卡片，而不是面试或数据分析
- 导航清楚表达“卡片是主线，知识库是来源，问答 / 诊断 / 分析是辅助”
- 用户可以稳定看到今日待学、今日待复习、今日完成率和连续学习天数

## 迭代建议

### 一期：记忆主线成型

- 建立卡片任务、首页主任务、复习中心和基础进度指标
- 打通知识库到卡片的生成链路
- 完成 SM-2 调度、掌握状态和卡片进度聚合

### 二期：理解增强

- 引入资料问答和引用溯源
- 让卡片学习后的疑问可以直接进入问答继续处理
- 补足资料管理、向量索引和检索体验

### 三期：诊断与复盘

- 引入面试诊断与历史复盘
- 增强数据分析、能力画像和进阶建议
- 在不破坏主线的前提下，把诊断和分析定位为辅助增强层
