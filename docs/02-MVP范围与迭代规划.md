# ByteCoach MVP 范围与迭代规划

## 1. 一期目标定义

一期目标不是做成完整商业产品，而是做出一个可运行、可展示、可扩展的 Java 面试智能学习 Agent MVP。

一期目标应满足以下标准：

- 有清晰的主业务闭环
- 有可展示的 AI 能力亮点
- 有完整的前后端工程结构
- 有合理的数据模型和接口设计
- 能支撑求职作品集和项目答辩

## 2. 一期必须完成的功能

### 2.1 用户与鉴权

- 用户注册
- 用户登录
- 获取当前用户信息
- 退出登录
- JWT 鉴权

### 2.2 首页 Dashboard

- 学习概览
- 最近面试结果
- 错题数量
- 薄弱知识点 Top N
- 学习计划完成率
- 快捷入口

### 2.3 Chat 问答

- 普通问答模式
- 知识库问答模式
- 会话列表
- 消息历史
- Markdown 回答展示
- 引用片段展示

### 2.4 知识库

- 内置 Java 面试资料
- 文档列表
- 分类查看
- 检索测试
- 后台导入与管理

### 2.5 模拟面试

- 选择面试方向
- 开始一场面试
- 获取当前题目
- 提交答案
- AI 评分
- AI 点评
- 单次追问
- 面试记录保存

### 2.6 错题本

- 自动记录低分题目
- 错题列表
- 错题详情
- 标准答案与错误原因查看
- 掌握状态更新
- 再次练习入口

### 2.7 学习计划

- 根据薄弱点和错题生成计划
- 查看计划
- 查看每日任务
- 更新任务完成状态

### 2.8 题库与后台管理

- 题目 CRUD
- 分类筛选
- 难度筛选
- 标准答案管理
- 文档管理
- 分类管理
- 后台前端复用同一个 Vue 应用，通过 `/admin/*` 路由承载管理入口

## 3. 一期可以简化的功能

- 用户画像只做派生统计
- 题目标签先用简单文本或弱关联
- 面试追问只做一轮
- 学习计划只做一次性生成
- RAG 不做 rerank 和多路召回
- 题目生成优先抽题，LLM 只做润色
- Qdrant 不作为阶段 1 启动必需依赖

## 4. 一期不建议做的功能

- Java 代码运行沙箱
- 知识图谱
- 用户长期记忆
- 多 Agent 编排
- 多轮复杂面试状态机
- 用户自助上传大文档并在线处理
- RAG 效果评估平台
- AI 评分效果评估平台

## 5. 二期规划

二期重点是增强产品真实可用性：

- 用户上传资料
- 文档异步解析、切分、向量化
- 面试报告增强
- 学习计划动态调整
- 用户画像增强
- Prompt 配置管理
- 模型切换能力
- RAG 召回与引用质量优化

## 6. 三期规划

三期重点是提升平台化和智能化能力：

- 长期记忆
- 多轮面试状态管理
- 评分效果评估体系
- 知识图谱辅助问答
- Java 代码运行沙箱
- 更复杂的 Agent 工具编排

## 7. 推荐交付顺序

建议按如下顺序开发：

1. 用户与鉴权
2. 公共能力与 AI 网关
3. 题库与知识库底座
4. Dashboard
5. Chat 问答
6. 面试模块
7. 错题本
8. 学习计划
9. 后台管理

## 8. 一期验收标准

满足以下条件即可视为一期完成：

- 用户可以完成从登录到问答或面试的主流程
- 面试后可以得到评分、点评和标准答案
- 低分题可以自动进入错题本
- 错题可驱动学习计划生成
- 首页可以看到学习状态变化
- 管理员可以维护题库、文档和分类
- 系统具备清晰的 AI 接入扩展位
