# 未来优化方向

> 基于当前项目状态，按优先级整理的可优化内容。不作为正式规划，仅作为备忘和参考。

---

## P0 — 安全与稳定性

### 密码重置

当前用户忘记密码后只能联系管理员手动重置。应支持：

- 邮箱验证码重置密码
- 需要集成邮件发送服务（SMTP / 第三方邮件 API）
- `user` 表已有 `email` 字段，基础设施就绪

### 接口幂等性

部分写操作（如提交答案、投票）缺少幂等保护，网络重试可能导致重复提交：

- 给关键写接口添加幂等 Token 机制
- 前端在请求时携带唯一 Token，后端通过 Redis 去重

### 文件上传安全

当前头像上传和知识库文档上传缺少文件类型校验的深度检查：

- 后端应校验文件 Magic Number（不仅依赖扩展名）
- 上传文件存储路径应与应用分离（当前存储在本地文件系统）
- 生产环境建议接入 OSS（阿里云/MinIO）

## P1 — 功能完善

### 密码找回 / 邮箱验证

- 注册时发送邮箱验证
- 忘记密码通过邮箱重置
- 需要引入邮件发送能力

### 社区内容富文本

当前社区提问和回答只支持纯文本，体验较差：

- 引入 Markdown 编辑器（如 md-editor-v3）
- 支持代码块高亮（面试回答经常包含代码）
- 支持图片上传

### 面试回放增强

当前面试详情页只展示文字记录：

- 语音面试支持音频回放（voice_record 表已有 audio_url）
- 支持逐题跳转和对比视图

### 学习提醒增强

当前通知系统只支持站内通知：

- 邮件提醒（复习提醒、计划提醒）
- 浏览器推送通知（Web Push API）
- 微信/钉钉 webhook 推送

### 知识库增强

- 支持更多文档格式（Word、PPT、HTML）
- 文档版本管理（当前只保留最新版本）
- 知识图谱可视化（知识点之间的关联关系）

## P2 — 性能优化

### 数据库优化

- **慢查询审计**：当前没有慢查询日志监控，生产环境应开启 `slow_query_log`
- **读写分离**：用户量增长后，可以将读操作分流到从库
- **分库分表**：`chat_message` 和 `review_log` 表数据增长快，可按用户 ID 分表
- **连接池调优**：HikariCP 参数（当前使用 Spring Boot 默认值）

### 缓存优化

- **Dashboard 缓存**：当前 TTL 5 分钟，可以改为事件驱动更新
- **分类缓存**：分类数据变化极少，可以长期缓存
- **用户信息缓存**：高频查询的用户信息可以缓存到 Redis

### 前端优化

- **代码分割**：当前所有页面都在一个 chunk，可以按路由懒加载进一步分割
- **图片优化**：头像图片缺少压缩和 CDN 加速
- **Service Worker**：添加离线缓存支持（当前有 OfflinePage 组件但未完整实现）
- **虚拟滚动**：社区问题列表、面试历史等长列表可以引入虚拟滚动

### LLM 调用优化

- **流式响应**：当前问答和面试评分是同步等待 LLM 返回，应改为 SSE 流式输出
- **Prompt 缓存**：相似的面试评分请求可以缓存 Prompt 结果
- **模型降级**：主模型不可用时自动切换到备选模型
- **异步队列**：批量操作（如计划生成）应放入消息队列异步处理

## P3 — 运维与监控

### 日志体系

- 当前日志直接输出到 stdout，生产环境应接入 ELK 或 Loki
- 结构化日志（JSON 格式）便于检索和分析
- 链路追踪（Trace ID）贯穿请求全链路

### 监控告警

- Prometheus + Grafana 指标采集（Actuator 已就绪，只需配置端点暴露）
- 自定义业务指标：LLM 调用成功率、面试完成率、复习完成率
- 告警规则：错误率 > 5%、响应时间 > 3s、磁盘 > 80%

### CI/CD

- GitHub Actions：PR 触发编译检查 + 单元测试
- main 分支合并触发 Docker 镜像构建 + 推送
- 自动部署到测试/生产环境

### 数据库迁移

- 当前使用手动 SQL 文件，应引入 Flyway 实现版本化迁移
- 避免多环境数据库结构不一致

## P4 — 体验优化

### 国际化

- 当前后端错误消息和前端 UI 全部硬编码中文
- 如需支持英文，需要引入 i18n 框架

### 移动端适配

- 当前有基本的响应式布局，但移动端体验仍有提升空间
- 考虑 PWA 支持（manifest.json + Service Worker）
- 语音面试在移动端体验更好，应重点优化

### 键盘快捷键

- 当前只有 Cmd+K 全局搜索
- 可以添加更多快捷键：跳转页面、快速操作等

### 数据可视化增强

- Dashboard 趋势图可以增加更多维度（按分类、按难度）
- 学习热力图（当前有 ReviewHeatmap，可以扩展到面试和问答）
- 能力雷达图（当前在自适应推荐中有能力画像，可以可视化）

### 社区增强

- 评论功能（当前只有回答，没有评论）
- @提及功能
- 问题标签系统（与题库标签联动）
- 精华帖/置顶帖
