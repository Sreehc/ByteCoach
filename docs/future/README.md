## 1. 目标定位

### 1.1 升级目标

OfferPilot V2 的主线从“今日卡片 + 复习闭环”切换为“AI Java 面试训练与求职管理闭环”。

目标版本围绕以下主线展开：

- 知识学习与题库训练
- RAG 知识库问答
- AI 模拟面试
- 学习计划与薄弱点分析
- 简历解析与项目问答助手
- 面试简历制作
- 岗位投递管理与 JD 分析
- 管理后台治理

### 1.2 目标用户

- 准备 Java 实习、校招、初级社招岗位的学习者
- 已有少量项目经验，但缺少系统化面试训练的开发者
- 需要同时管理学习资料、面试准备、简历打磨和投递流程的求职者

### 1.3 新版产品心智

- 用户进入首页后，主任务不再是“先完成今日卡片”，而是进入“求职训练工作台”
- 题库、知识库、模拟面试、学习计划、简历、投递形成统一求职准备闭环
- 卡片与复习继续保留，但只作为学习计划和薄弱点改进的辅助能力

### 1.4 本轮冻结模块

以下模块保留现状，不作为本轮升级重点：

- 社区 / 排行榜：不调整功能边界与产品定位
- 账户安全增强：现有 2FA、设备管理、登录日志继续保留，本轮不新增更重的安全功能
- 卡片 / 复习：保留现有实现，但不再作为产品叙事主轴

## 3. 关键差异说明

### 3.1 题库 vs 知识库重构差异

#### 当前边界

- `question` 更偏基础题库实体，重点在结构化题目、标准答案、评分标准
- `knowledge_doc` / `knowledge_chunk` 更偏资料存储、切分和向量检索
- 知识库当前同时承担“RAG 语料来源”和“卡片来源”两种职责

#### 目标边界

- 题库负责结构化训练内容：
  - 题目标题
  - 技术分类
  - 难度
  - 标签
  - 高频程度
  - 适用岗位方向
  - 标准答案
  - 面试版回答
  - 追问建议
  - 常见错误回答
- 知识库负责非结构化语料中心：
  - 系统知识资料
  - 用户上传资料
  - 简历资料
  - 岗位 JD
  - 项目拆解文档
  - 后续可扩展的 Word/PPT/HTML 内容

#### 重构原则

- 题库和知识库不是重复模块
- 题库提供“结构化训练内容”
- 知识库提供“非结构化检索语料”
- RAG、模拟面试、学习计划、简历问答都可以消费知识库
- 卡片模块未来只从题库、知识库、面试错题中选择性取材，不再反向定义知识库定位

### 3.2 RAG 知识库差异

#### 当前能力

- 支持文档解析、切分、Embedding、Redis 向量检索
- 支持带引用的知识问答
- 支持后台重切分 / 重建索引的基础治理接口

#### 目标能力

- 区分系统知识库与个人知识库
- 支持 Markdown / TXT / PDF / Word，保留 PPT / HTML 扩展位
- 支持回答模式切换：
  - 学习版
  - 面试版
  - 简短版
  - 项目结合版
- 支持引用来源追溯和追问模板
- 支持简历、JD、项目文档作为 RAG 语料输入
- 支持后台对文档解析状态、切分状态、向量化状态进行治理

### 3.3 卡片与复习降级说明

- 当前 `knowledge_card_task`、`knowledge_card`、`knowledge_card_log`、`wrong_question`、`review_log` 等模型保留
- 首页、导航、阶段优先级、版本叙事不再围绕卡片组织
- 卡片与复习未来的角色：
  - 承接学习计划中的专项巩固任务
  - 承接模拟面试后的薄弱点复习
  - 承接知识库或题库中的重点内容强化
- 本轮不继续推进“卡片即主线”的产品路线

## 7. 风险与依赖

### 7.1 产品风险

- 需求范围扩大后，容易再次演化成“大而全平台”
- 如果首页和导航不及时切换，用户仍会把产品理解为“卡片学习工具”
- 简历、投递、学习计划三条新链路如果分阶段落地不清晰，容易形成半成品入口

### 7.2 数据风险

- 现有数据库文档与实际建表之间存在不完全一致处，V2 前需要重新对齐
- 新增多表后，旧业务和新业务的数据归属需要统一口径
- 简历、JD、投递记录都包含敏感内容，必须补齐访问控制与脱敏策略

### 7.3 技术风险

- RAG 范围扩大后，文档来源和向量治理复杂度显著提升
- 简历解析和 JD 解析对文档质量依赖高，解析失败兜底必须明确
- 面试、问答、计划、简历、投递都会消费 LLM，AI 成本和调用稳定性要纳入后台治理

### 7.4 实施依赖

- 首页与导航重构应先于功能大规模扩展
- 文件存储方案需要早于简历、知识库格式扩展和语音文件治理落地
- 题库 / 知识库边界厘清需要早于 RAG、简历、JD 模块扩展
- 学习计划与薄弱点模型应早于卡片辅助联动定义
