# JVM 内存区域

JVM 运行时内存区域通常分为堆、虚拟机栈、本地方法栈、程序计数器和方法区。堆是垃圾回收的主要区域，栈负责保存方法调用帧。

# 垃圾回收器分类

JVM 垃圾回收器分类可以按新生代、老年代和整堆回收来理解。Serial、ParNew、Parallel Scavenge 常见于新生代；CMS 和 Serial Old 常见于老年代；G1 支持按 Region 做整堆回收。

## CMS 与 G1

CMS 以低停顿为目标，但会带来内存碎片和并发失败问题。G1 把堆划分为多个 Region，通过预测停顿时间控制回收节奏，更适合大堆场景。

# 类加载

类加载过程包括加载、验证、准备、解析和初始化。双亲委派模型是高频面试点，常用于解释类隔离与安全边界。
